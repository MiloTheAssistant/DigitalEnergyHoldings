## Balanced Trinity

DEH visual continuity comes from a balanced three-lane system. Future assets should clearly belong to one of these lanes, or intentionally blend two of them.

### Boardroom Vault

Use for ownership, governance, contracts, IP, legal clarity, banking trust, asset control, and long-term stewardship.

Visual cues:
- Vault doors, secure thresholds, black glass, brushed metal, polished stone.
- Warm gold light, architectural symmetry, controlled reflections.
- Quiet authority rather than luxury excess.

### Command Center

Use for AI systems, automation rails, compute, workflow orchestration, operating intelligence, and shared services.

Visual cues:
- Executive command rooms, abstract system maps, server architecture, glass walls.
- Restrained cyan signal lines paired with gold control lighting.
- Operational clarity without busy dashboards or fake interface text.

### Disciplined Future

Use for digital energy infrastructure, governed digital assets, treasury discipline, frontier strategy, and long-term capital posture.

Visual cues:
- Data-center campuses, energy-aware infrastructure, St. Louis skyline hints, high-trust future architecture.
- Night scenes, measured cyan pathways, dark negative space, grounded realism.
- Future-facing without hype, coins, tickers, or speculative visual noise.

## Anti-Slop Standard

Artistic exploration is welcome inside the Balanced Trinity. Technical execution should stay close to the path.

Every accepted image must have:
- A clear Trinity lane.
- A defined use case.
- A reason to exist in the brand system.
- A coherent relationship to the current source family.
- Enough restraint to feel premium rather than generated.

Avoid:
- Fake readable UI text.
- Watermarks or generated logos.
- Cluttered stock-photo energy.
- Random technology props.
- Cryptocurrency coins, stock tickers, hype charts, or humanoid robots.
- Unrelated scenes that only look "cool" but do not support DEH.

## Quality Rules

- Photorealistic, premium, and architectural.
- No fake readable UI text.
- No logos in generated scenes.
- No watermarks.
- No people as the main subject except anonymous/background human presence in the profile stamp.
- No identifiable portraits unless explicitly approved for a specific campaign.
- No cluttered stock-photo feel.
- Safe negative space for overlays and platform UI chrome.
- Small source families are preferred over unrelated one-off scenes.
- Platform files should be derived from shared masters when formats overlap.

## Prompt Framework

Use this DEH-specific prompt framework when generating a new brand-image family.

```text
I need to create a complete photoreal brand image family for Digital Energy Holdings.

Project:
- Website/project name: Digital Energy Holdings
- Website domain: https://digitalenergyholdings.com
- Business/location/audience: Holding-company asset and systems layer based in St. Louis, MO, serving bankers, CPAs, vendors, partners, operating-company stakeholders, and counterparties who need a clear, trustworthy explanation of the structure.
- Current hero image/vibe: Dark premium vault and infrastructure scene with black glass, brushed metal, warm gold light, restrained cyan signal lines, and safe negative space.
- Brand tone: disciplined, premium, governed, technical, future-aware, human-centered only when anonymous and controlled.
- Brand colors and visual cues: #050608 black base, slate/graphite materials, warm metallic gold, restrained cyan signal accents, polished stone, secure thresholds, command-center architecture, data infrastructure.

Goal:
Create project-specific image assets that complement the existing hero and carry the Balanced Trinity forward:
- Boardroom Vault for ownership, governance, contracts, IP, and trust.
- Command Center for AI systems, compute, automation rails, and operations.
- Disciplined Future for governed digital assets, energy infrastructure, and long-term frontier posture.

Required source family:
1. A main human-centered profile stamp that works as the primary profile picture.
2. A clean wide photoreal background family for website, Facebook, LinkedIn, X, and feed/link assets.
3. A clean vertical photoreal background family for Instagram and TikTok.

Direction:
Use a Hybrid Brand Library approach:
- One strong human-centered profile stamp.
- A related family of platform-native photoreal backgrounds.
- Keep the assets visually unified.
- Derive platform-specific files from shared masters where formats overlap.

Profile stamp rules:
- Human-centered brand mark, not just an abstract background.
- Anonymous human presence only: back view, silhouette, reflection, or non-identifiable figure.
- Circular-crop safe.
- Strong at small profile-picture sizes.
- No fake words, watermarks, slogans, detailed typography, or readable UI.
- Should feel like a premium dimensional brand stamp, not a generic stock logo.

Background rules:
- Clean photoreal backgrounds.
- No embedded text, logos, slogans, or watermarks.
- Leave negative space for future overlays.
- Use real-world scenes connected to DEH: secure asset infrastructure, command environments, data centers, St. Louis hints, governance, AI systems, and digital energy.
- Avoid identifiable portraits, distorted hands, clutter, fake readable UI text, crypto coins, stock tickers, humanoid robots, and generic stock-photo energy.

After generation:
- Save final PNGs into the platform folders.
- Validate that every file exists, opens as an image, has nonzero size, and has sensible platform dimensions.
- Create or update this README inventory.
- Do not overwrite the existing hero unless explicitly asked.
```

## Approval Checklist

Before accepting any new DEH brand image, answer these questions:

- Which Trinity lane does this serve: Boardroom Vault, Command Center, Disciplined Future, or an intentional blend?
- Where will this image be used?
- Does the image support DEH's holding-company role?
- Is it photorealistic, premium, and restrained?
- Does it avoid fake readable UI text?
- Does it avoid generated logos and watermarks?
- Does it avoid visual clutter and generic stock-photo energy?
- Does it preserve safe negative space for overlays or platform UI?
- If a human appears, are they anonymous, non-identifiable, and not the main subject except for profile-stamp use?
- Is the file named for its platform and use case?
- Does the exported image match the expected dimensions?
- Is it derived from the current source family, or is there a documented reason to create a new family?

If any answer is weak, revise before adding the asset to the brand library.
